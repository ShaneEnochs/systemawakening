*comment ─── CHECKPOINT PROCEDURE ───
*comment Called via *call checkpoint after each cluster.
*comment Reads meters and prestige, determines branch_state.
*comment The calling tier file handles routing based on branch_state.

*procedure checkpoint

  *comment Step 1: Meter collapse check (instant death)
  *if people <= 0
    *set branch_state "dead"
    *set pending_event "fail_people_low"
    *return
  *if people >= 100
    *set branch_state "dead"
    *set pending_event "fail_people_high"
    *return
  *if treasury <= 0
    *set branch_state "dead"
    *set pending_event "fail_treasury_low"
    *return
  *if treasury >= 100
    *set branch_state "dead"
    *set pending_event "fail_treasury_high"
    *return
  *if church <= 0
    *set branch_state "dead"
    *set pending_event "fail_church_low"
    *return
  *if church >= 100
    *set branch_state "dead"
    *set pending_event "fail_church_high"
    *return
  *if nobility <= 0
    *set branch_state "dead"
    *set pending_event "fail_nobility_low"
    *return
  *if nobility >= 100
    *set branch_state "dead"
    *set pending_event "fail_nobility_high"
    *return

  *comment Step 2: Chronic failure check
  *if fail_count >= 3
    *set branch_state "dead"
    *set pending_event "fail_chronic"
    *return

  *comment Step 3: Failing — any meter below 10, or two meters below 25
  *if (people < 10) or (treasury < 10) or (church < 10) or (nobility < 10)
    *set branch_state "failing"
    *set fail_count +1
    *return

  *if (people < 25) and (treasury < 25)
    *set branch_state "failing"
    *set fail_count +1
    *return
  *if (people < 25) and (church < 25)
    *set branch_state "failing"
    *set fail_count +1
    *return
  *if (people < 25) and (nobility < 25)
    *set branch_state "failing"
    *set fail_count +1
    *return
  *if (treasury < 25) and (church < 25)
    *set branch_state "failing"
    *set fail_count +1
    *return
  *if (treasury < 25) and (nobility < 25)
    *set branch_state "failing"
    *set fail_count +1
    *return
  *if (church < 25) and (nobility < 25)
    *set branch_state "failing"
    *set fail_count +1
    *return

  *comment Step 4: Struggling — any meter below 25 or above 85
  *if (people < 25) or (treasury < 25) or (church < 25) or (nobility < 25)
    *set branch_state "struggling"
    *return
  *if (people > 85) or (treasury > 85) or (church > 85) or (nobility > 85)
    *set branch_state "struggling"
    *return

  *comment Step 5: Advancing — prestige meets threshold and all meters healthy
  *temp threshold 9999
  *if tier = "baron"
    *set threshold 30
  *if tier = "viscount"
    *set threshold 70
  *if tier = "count"
    *set threshold 130
  *if tier = "duke"
    *set threshold 220
  *if tier = "archduke"
    *set threshold 340
  *if tier = "prince"
    *set threshold 500
  *if tier = "king"
    *set threshold 700

  *if (prestige >= threshold) and (people >= 30) and (people <= 80) and (treasury >= 30) and (treasury <= 80) and (church >= 30) and (church <= 80) and (nobility >= 30) and (nobility <= 80)
    *set branch_state "advancing"
    *return

  *comment Step 6: Stagnating — all meters healthy but prestige too low
  *if (people >= 30) and (people <= 80) and (treasury >= 30) and (treasury <= 80) and (church >= 30) and (church <= 80) and (nobility >= 30) and (nobility <= 80)
    *set branch_state "stagnating"
    *return

  *comment Step 7: Thriving — default
  *set branch_state "thriving"
  *return


*comment ─── PROMOTE PROCEDURE ───
*comment Called via *call promote when the player advances to the next tier.
*comment Updates the tier string, title (pronoun-aware), and resets per-tier counters.

*procedure promote

  *if tier = "baron"
    *set tier "viscount"
    *if pronouns_subject = "he"
      *set title "Viscount"
    *elseif pronouns_subject = "she"
      *set title "Viscountess"
    *else
      *set title "Viscount/ess"
    *set fail_count 0
    *set clusters_this_tier 0
    *return

  *if tier = "viscount"
    *set tier "count"
    *if pronouns_subject = "he"
      *set title "Count"
    *elseif pronouns_subject = "she"
      *set title "Countess"
    *else
      *set title "Count/ess"
    *set fail_count 0
    *set clusters_this_tier 0
    *return

  *if tier = "count"
    *set tier "duke"
    *if pronouns_subject = "he"
      *set title "Duke"
    *elseif pronouns_subject = "she"
      *set title "Duchess"
    *else
      *set title "Duke/Duchess"
    *set fail_count 0
    *set clusters_this_tier 0
    *return

  *if tier = "duke"
    *set tier "archduke"
    *if pronouns_subject = "he"
      *set title "Archduke"
    *elseif pronouns_subject = "she"
      *set title "Archduchess"
    *else
      *set title "Archduke/ess"
    *set fail_count 0
    *set clusters_this_tier 0
    *return

  *if tier = "archduke"
    *set tier "prince"
    *if pronouns_subject = "he"
      *set title "Prince"
    *elseif pronouns_subject = "she"
      *set title "Princess"
    *else
      *set title "Prince/ss"
    *set fail_count 0
    *set clusters_this_tier 0
    *return

  *if tier = "prince"
    *set tier "king"
    *if pronouns_subject = "he"
      *set title "King"
    *elseif pronouns_subject = "she"
      *set title "Queen"
    *else
      *set title "Monarch"
    *set fail_count 0
    *set clusters_this_tier 0
    *return

  *if tier = "king"
    *set tier "emperor"
    *if pronouns_subject = "he"
      *set title "Emperor"
    *elseif pronouns_subject = "she"
      *set title "Empress"
    *else
      *set title "Emperor/Empress"
    *set fail_count 0
    *set clusters_this_tier 0
    *return

  *return
